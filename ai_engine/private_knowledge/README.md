Drop your private trading documents here before rebuilding the knowledge base.

Supported now:
- .txt
- .md
- .pdf (requires `pypdf` to be installed in the project venv)

Recommended ingestion workflow:
1. Put only stable private knowledge here, such as trading rules, playbooks, and research frameworks.
2. Keep one topic per file when possible.
3. Clean obvious boilerplate before importing.
4. Rebuild the knowledge base through the API after adding or updating files.

Examples:
- private_knowledge/entry_rules.txt
- private_knowledge/risk_management.txt
- private_knowledge/playbooks/gap_breakout.md
